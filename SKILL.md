# Скилл: Яндекс Wordstat

Собирает частотности и похожие запросы из Wordstat через `agent-browser`.
Всю навигацию и парсинг делает Claude вживую по snapshot-дереву.

## Использование

```
/yandex-wordstat <фразы> [--регион|--region <город>] [--тема <слаг>] [--out <csv>]
```

- **фразы** (обязательно) — одна или несколько базовых фраз. Разделитель `;` или перенос строки.
- **--регион / --region** — название региона как в Wordstat. Дефолт: **Беларусь**.
- **--тема** — слаг для имени папки результата. Дефолт: слаг из первой фразы.
- **--out** — путь к CSV. Дефолт: `<корень скилла>/research/<YYYY-MM-DD>-<тема>/keywords.csv`.

Пример: `/yandex-wordstat ателье штор; пошив штор на заказ --регион Минск`

## Бинарь

```
agent-browser
```
Должен быть в `PATH`. Проверка: `agent-browser --help`. Если нет —
`npm i -g agent-browser`.

## Подготовка

1. Разобрать аргументы: список фраз, регион (дефолт «Беларусь»), тему, путь out.
2. Тема-слаг: из `--тема`, иначе из первой фразы — нижний регистр, пробелы → дефисы,
   кириллица сохраняется (например «ателье штор» → `ателье-штор`).
3. Получить дату для имени папки: спросить у системы — `date +%Y-%m-%d`
   (НЕ выдумывать дату из головы).
4. Каталог результата: `<корень скилла>/research/<дата>-<тема>/`. Создать:
   `mkdir -p "<каталог>"`.

## Шаг 1. Открыть Wordstat в именованной сессии

```bash
agent-browser --session yandex open https://wordstat.yandex.ru
agent-browser --session yandex wait --load networkidle
agent-browser --session yandex snapshot -i
```

Сессия `yandex` переживает между запусками — auth-стейт лежит в `~/.agent-browser/`.

## Шаг 2. Проверка авторизации и капчи

По snapshot определить состояние:
- Видна форма/кнопка «Войти» → НЕ залогинен.
- Видна капча / «Подтвердите, что вы не робот» → капча.
- Видны поле ввода запроса и данные → всё ок, перейти к Шагу 3.

Если НЕ залогинен или капча:
1. Сообщить Анатолю: «Wordstat просит войти/капчу. Открываю окно браузера —
   залогинься руками».
2. Открыть видимое окно той же сессии:
   ```bash
   agent-browser --session yandex --headed open https://passport.yandex.ru
   ```
3. Дождаться подтверждения от Анатоля, что он вошёл.
4. Вернуться на Wordstat, re-snapshot, убедиться что данные доступны.

НИКОГДА не вводить логин/пароль самостоятельно и не пытаться обойти капчу.

## Шаг 3. Установить регион

1. Re-snapshot, найти переключатель региона (по умолчанию обычно «Все регионы»).
2. Кликнуть его, в поле поиска региона ввести значение (дефолт «Беларусь»),
   выбрать из списка, применить.
3. Если регион не найден в списке — НЕ подставлять другой молча: сообщить Анатолю
   и спросить корректное название.
4. Re-snapshot после применения.

## Шаг 4. Сбор по каждой фразе

Для каждой базовой фразы из списка:

1. Найти поле ввода запроса (re-snapshot), очистить, ввести фразу, нажать Enter / «Подобрать»:
   ```bash
   agent-browser --session yandex snapshot -i
   agent-browser --session yandex fill @<ref-поля> "<фраза>"
   agent-browser --session yandex press "Enter"
   agent-browser --session yandex wait --load networkidle
   ```
2. Re-snapshot. Прочитать **базовую частотность** (общее число показов для фразы).
3. Снять **точную частотность**: повторить ввод с оператором — `"!<фраза>"`
   (каждое слово с `!` при необходимости, начни с фразового оператора `"!фраза"`),
   прочитать число.
4. Прочитать блок **«Запросы, похожие на …»** (правая колонка): собрать пары
   «фраза → частотность».
5. Если по фразе пусто/0 — записать строку с пустой частотностью, продолжить
   (не падать).

Между фразами небольшая пауза (`agent-browser --session yandex wait 1500`),
чтобы не словить лимит.

## Шаг 5. Записать результат

### keywords.csv

Колонки: `базовая_фраза,ключевая_фраза,тип,частотность,регион`
- `тип` ∈ {`базовая`, `точная`, `похожая`}
- UTF-8, разделитель — запятая, поля с запятыми — в двойных кавычках.

Записать через инструмент Write (не через echo).

### meta.md

```
# Wordstat выгрузка

- Дата: <дата сбора>
- Регион: <регион>
- Исходные фразы: <список>
- Всего ключей: <число строк>
```

## Шаг 6. Сводка в чат

Показать Markdown-таблицу: по каждой базовой фразе — базовая и точная частотность
и число найденных похожих. В конце — путь к сохранённому CSV.

## Обработка ошибок

| Ситуация | Действие |
|----------|----------|
| Не залогинен / капча / протухшая сессия | Пауза, headed-окно, просьба к Анатолю войти руками, продолжить |
| Пустой результат по фразе | Пустая частотность в CSV, идти дальше |
| Регион не найден | Спросить корректное название, не подставлять молча |
| `agent-browser` не найден | Подсказать `npm i -g agent-browser` |

## Чего НЕ делать

- Не хранить и не вводить пароли; вход — только руками.
- Не обходить капчу автоматически.
- Не уходить в авто-расширение (похожие к похожим) — только то, что просили.
- Не коммитить `research/` — она в `.gitignore`.
