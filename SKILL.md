# Скилл: Яндекс Wordstat

Собирает частотности и связанные запросы из Wordstat через `agent-browser`.
Навигация — **через URL** (`?words=<фраза>&region=<id>`), это надёжнее ввода в
поле и кликов по модалу региона. Чтение данных — по snapshot-дереву.

## Использование

```
/yandex-wordstat <фразы> [--регион|--region <город>] [--тема <слаг>] [--out <csv>]
```

- **фразы** (обязательно) — одна или несколько базовых фраз. Разделитель `;` или перенос строки.
- **--регион / --region** — название региона. Дефолт: **Беларусь**. Маппинг названия → id см. «Таблица регионов».
- **--тема** — слаг для имени папки результата. Дефолт: слаг из первой фразы.
- **--out** — путь к CSV. Дефолт: `<корень скилла>/research/<YYYY-MM-DD>-<тема>/keywords.csv`.

Пример: `/yandex-wordstat ателье штор; пошив штор на заказ --регион Минск`

## Бинарь

`agent-browser` должен быть в `PATH` (проверка: `agent-browser --help`; если нет — `npm i -g agent-browser`).
`node` тоже нужен (идёт вместе с npm) — для URL-кодирования кириллицы.

## Таблица регионов (geo-id)

Подтверждённые id (проверены вживую):

| Регион | id |
|--------|-----|
| Беларусь (дефолт) | 149 |
| Минск | 157 |
| Москва | 213 |
| Россия | 225 |

Для региона не из таблицы — узнать id один раз вручную: открыть
`https://wordstat.yandex.ru` headed, сделать любой запрос, нажать кнопку региона
(«Все регионы»), выбрать нужный регион в дереве, **Подтвердить** — id появится в
адресной строке как `?region=<id>`. Записать его в эту таблицу.
Не подставлять регион молча, если id неизвестен — спросить Анатоля.

## Подготовка

1. Разобрать аргументы: список фраз, регион (дефолт «Беларусь» → id 149), тему, путь out.
2. Определить `region_id` по «Таблице регионов».
3. Тема-слаг: из `--тема`, иначе из первой фразы — нижний регистр, пробелы → дефисы, кириллица сохраняется.
4. Дата папки: `date +%Y-%m-%d` (НЕ выдумывать из головы).
5. Каталог: `mkdir -p "<корень скилла>/research/<дата>-<тема>/"`.

## Шаг 1. Проверка авторизации (один раз)

```bash
agent-browser --session yandex open "https://wordstat.yandex.ru/?words=%D1%82%D0%B5%D1%81%D1%82&region=149"
agent-browser --session yandex wait --load networkidle
agent-browser --session yandex snapshot -i
```

По snapshot:
- Есть ссылка «Выйти» и таблица с числами → залогинен, перейти к Шагу 2.
- Видна форма «Log in with ID» / редирект на passport → НЕ залогинен.
- Капча / «Подтвердите, что вы не робот» → капча.

Если НЕ залогинен или капча:
1. Сообщить Анатолю: «Wordstat просит войти/капчу. Открываю окно браузера — залогинься руками».
2. `agent-browser --session yandex --headed open "https://passport.yandex.ru"`
3. Дождаться подтверждения от Анатоля, что он вошёл.
4. Re-snapshot Wordstat, убедиться что данные доступны.

При первом заходе бывает оверлей «Тур по интерфейсу» — закрыть (кнопка «Close Tour» / крестик), если мешает чтению.

НИКОГДА не вводить логин/пароль самостоятельно и не пытаться обойти капчу.

## Шаг 2. Сбор по каждой фразе

Для каждой базовой фразы. URL-кодировать фразу (кириллица!) через node:

```bash
enc=$(node -e 'process.stdout.write(encodeURIComponent(process.argv[1]))' "<фраза>")
```

### 2a. Базовая частотность + «Запросы со словами» (режим «Популярные»)

```bash
agent-browser --session yandex open "https://wordstat.yandex.ru/?words=$enc&region=<region_id>"
agent-browser --session yandex wait --load networkidle
agent-browser --session yandex snapshot -i
```

Прочитать таблицу `cell`-ов (пары «фраза → число»):
- **Первая строка** = сама фраза с её числом → это **базовая частотность** (тип `базовая`).
- **Все строки** таблицы = запросы, содержащие фразу → тип `со_словами` (это ядро семантики).
- Если таблица длинная и видна не вся — проскроллить (`scroll down`) и re-snapshot, добрать строки.

### 2b. Точная частотность (оператор-кавычки)

```bash
enc2=$(node -e 'process.stdout.write(encodeURIComponent(process.argv[1]))' "\"<фраза>\"")
agent-browser --session yandex open "https://wordstat.yandex.ru/?words=$enc2&region=<region_id>"
agent-browser --session yandex wait --load networkidle
agent-browser --session yandex snapshot -i
```

Первая строка таблицы = **точная частотность** фразы (тип `точная`).

### 2c. Похожие запросы (режим «Похожие»)

Вернуться к виду 2a (или с того же экрана), переключить radio «Похожие»:

```bash
agent-browser --session yandex snapshot -i        # найти ref у label "Похожие"
agent-browser --session yandex click @<ref-Похожие>
agent-browser --session yandex wait --load networkidle
agent-browser --session yandex snapshot -i
```

Прочитать таблицу → пары «фраза → число», тип `похожая`.

Между фразами пауза: `agent-browser --session yandex wait 1500` (чтобы не словить лимит).
Если по фразе пусто/0 — записать строку с пустой частотностью, не падать.

## Шаг 3. Записать результат

### keywords.csv

Колонки: `базовая_фраза,ключевая_фраза,тип,частотность,регион`
- `тип` ∈ {`базовая`, `точная`, `со_словами`, `похожая`}
- UTF-8, разделитель — запятая, поля с запятыми — в двойных кавычках.
- Писать через инструмент Write (не через echo).

### meta.md

```
# Wordstat выгрузка

- Дата: <дата>
- Регион: <регион> (geo-id <id>)
- Исходные фразы: <список>
- Всего ключей: <число строк>
```

## Шаг 4. Сводка в чат

Markdown-таблица: по каждой базовой фразе — базовая и точная частотность, число
«со словами» и число похожих. В конце — путь к сохранённому CSV.

## Обработка ошибок

| Ситуация | Действие |
|----------|----------|
| Не залогинен / капча / протухшая сессия | Пауза, headed-окно passport, просьба к Анатолю войти руками, продолжить |
| Пустой результат по фразе | Пустая частотность в CSV, идти дальше |
| Регион не в таблице | Узнать id вручную (см. «Таблица регионов») или спросить Анатоля; не подставлять молча |
| `agent-browser`/`node` не найден | Подсказать `npm i -g agent-browser` |

## Чего НЕ делать

- Не хранить и не вводить пароли; вход — только руками.
- Не обходить капчу автоматически.
- Не уходить в авто-расширение (похожие к похожим) — только то, что просили.
- Не коммитить `research/` — она в `.gitignore`.
